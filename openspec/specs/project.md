# 项目背景

## 组织简介

BSR 是一个救援组织，专门处理城市中的紧急救援事件。组织公布了紧急联系电话，接收求助后通过 IM 平台协调救援活动。

## 业务场景

当接到紧急求助电话后，救援组织者会在 IM 平台发布救援事件信息，包括：
- 事件描述（如：某地有一只猫被困）
- 地点（地址和地图坐标）
- 招募救援人员数量
- 所需技能要求（如：绳索攀爬能力）
- 招募截止时间

队员看到救援事件后，可在 IM 平台报名，提供：
- 姓名
- 参与人数
- 当前位置和预计到达时间

## 项目目标

为提高救援信息发布和响应效率，BSR 需要建立"应急事件发布响应系统"（Emergency Response System，简称 ERS）。

## 系统组成

ERS 由两个核心模块组成：

### 1. 组织管理系统（Web 程序）
功能包括：
- 发布紧急救援信息，招募救援人员
- 查看参与救援的人员、位置和到达时间
- 查看救援人员在活动中发送的信息（文字、图片、视频），了解救援进展
- 结束救援活动，发出总结报告
- 查看救援人员的服务时间统计

### 2. 救援报名系统（移动端程序）
功能包括：
- 接收紧急救援信息推送
- 查看救援详情，决定是否参加
- 参与救援期间实时发送位置信息
- 救援活动期间发送信息（文字、图片、视频）
- 救援结束后查看个人服务时间记录

## 技术栈

- 管理子系统前端: React
- 管理子系统后端: Sprint Boot
- 执行子系统：Expo (一份代码支持 Android 和 iOS 平台)
- 数据库：MySQL 8
- 缓存：Redis
- 消息队列：RabbitMQ
- 对象存储：MinIO
- 用户认证：Keycloak（OIDC 协议）
- 推送服务：极光平台
- 地图服务：百度地图

# 代码规范

- TypeScript 编码规范
- Conventional Commits specification

# 约束条件

- 系统在自建服务器部署
- 基础组件和自研组件都使用 Docker 镜像部署，使用 docker-compose 管理容器
- 移动端参与任务期间定期上报 GPS 位置（每 30 秒上报一次）
- Redis 缓存最新位置数据，减少数据库压力
- 图片自动压缩，生成缩略图
